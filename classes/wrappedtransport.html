  • disconnectPromise(options?: any): Promise<any>

  • Overrides Transport.disconnectPromise

    • Defined in ua.ts:124

    Disconnect socket. It could happen that the user switches network interfaces while calling. If this happens, closing a websocket will cause it to be blocked. To make sure that UA gets to the proper internal state so that it is ready to 'switch over' to the new network interface with a new websocket, we call the function that normally causes the disconnectPromise to be resolved after a timeout.
